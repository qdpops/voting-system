# True Vote — Система электронного голосования

Веб-приложение для проведения закрытых голосований с персональными ссылками и уведомлениями по email.

---

## Возможности

- **Панель администратора** — управление участниками и голосованиями
- **Персональные ссылки** — каждый участник получает уникальную одноразовую ссылку на email
- **Защита от повторного голосования** — токен сгорает после использования
- **Три варианта ответа** — «За», «Против», «Воздержался»
- **Статистика в реальном времени** — результаты с кворумом доступны сразу после голосования
- **Черновики** — голосование можно подготовить заранее и запустить в нужный момент
- **Docker-готовность** — разворачивается одной командой

---

## Стек

| Компонент | Технология |
|-----------|-----------|
| Backend | [FastAPI](https://fastapi.tiangolo.com/) |
| База данных | SQLite |
| Шаблоны | Jinja2 |
| Email | SMTP (smtplib) |
| Контейнеризация | Docker + docker-compose |

---

## Быстрый старт

### Локально

```bash
pip install -r requirements.txt
uvicorn main:app --reload
```

Откройте [http://localhost:8000/admin](http://localhost:8000/admin)  
Токен по умолчанию: `supersecret123`

### Docker

```bash
cp .env.example .env
# Отредактируйте .env — укажите ADMIN_TOKEN и SMTP_PASS
docker compose up -d
```

## Переменные окружения

Скопируйте `.env.example` в `.env` и заполните:

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `ADMIN_TOKEN` | Токен доступа к панели администратора | `supersecret123` |
| `DB_PATH` | Путь к файлу базы данных | `voting.db` |
| `SMTP_SERVER` | SMTP-сервер для отправки писем | `smtp.jino.ru` |
| `SMTP_PORT` | Порт SMTP | `587` |
| `SMTP_USER` | Логин почтового ящика | — |
| `SMTP_PASS` | Пароль почтового ящика | — |

## Сценарий использования

```
1. Добавьте участников (организация, номер, email)
2. Создайте черновик голосования — укажите вопрос и длительность
3. Нажмите «Запустить» — ссылки автоматически разойдутся участникам
4. Участники голосуют по персональной ссылке из письма
5. Результаты и кворум доступны в разделе «Результаты»
```

## Маршруты API

| Метод | URL | Описание |
|-------|-----|----------|
| `GET` | `/admin` | Панель администратора |
| `POST` | `/admin/add_user` | Добавить участника |
| `POST` | `/admin/delete_user` | Удалить участника |
| `POST` | `/admin/create_voting` | Создать черновик голосования |
| `POST` | `/admin/launch_voting` | Запустить голосование и разослать ссылки |
| `POST` | `/admin/edit_voting` | Редактировать черновик |
| `POST` | `/admin/delete_voting` | Удалить голосование |
| `GET` | `/vote/{id}/{token}` | Страница голосования |
| `POST` | `/vote/{id}/{token}` | Отправить голос |
| `GET` | `/results/{id}` | Результаты голосования |
| `GET` | `/docs` | Swagger UI |
